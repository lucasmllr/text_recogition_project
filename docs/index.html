Document Recognition Project Documentation¶

This is the documentation for the optical character recognition project hosted here. It was created as a student project. For details on the used algorithms and results please refer to this report. For an overview of the code structure see further below.

Code structure:¶

The code is structured as follows:

At the top is the pipeline.py script whose function read() takes as input an image, runs the segmentation, rotation and classification algorithms on it and returns a list of strings as result.

interface.py provides a terminal interface for the read() function.

arguments.py holds a class defining all parameters of the program.

blob_extraction.py contains the simple connected component analysis algorithm to propose letter candidates.

mser_extraction.py is the more advances maximally stable extremal regions extractor to propose letter candidates.

components.py contains classes to store character candidates proposed by either of the above two segmentation algorithms. All subsequent operations are done on an object of the Components class.

component_evaluation.py contains functions to work on the Components class including the heuristic filtering of character candidates.

bbox_based_rot_correction.py is the faster, character-based rotation algorithm

rot_correction.py is the slower pixel-based rotation algorithm.

line_extraction.py is the projection based line segmentation algorithm that is only used in combination with the pixel based rotation.

model.py holds the architecture of the CNN used to classify character candidates.

train_nn.py contains functions to train and test the CNN for classification.

data_gen.py includes all methods to generate our artificial images.

detect_iban.py contains methods to find an IBAN in a string.

test.py runs the read() function on a directory containing jpgs and prints extracted IBANs to the console.

evaluation.ipynb visualizes the proceeding of the full algorithm for an example image.
